# voicelab 평가 모듈 — 작업 인수인계 (Claude Code용)

> 이 문서 + `voicelab_eval_module.jsx`를 함께 레포에 넣고 작업을 이어가세요.
> 한 줄 요약: **지산학(산학협력) 시범사업에서 어르신의 인지·언어·정서·삶의 질을 사전/중간/사후로 평가하는 검사자-실시형 웹앱 모듈.** voicelab 허브에 들어갈 예정.

---

## 1. 프로젝트 개요

- **사업**: 대림대학교 언어치료학과(허브) · ㈜마인드허브 · 안양시노인종합복지관 협력 시범사업
- **대상**: 지역사회 **정상 인지 어르신** 10~14명 (진단이 아니라 **변화 추적**이 목적)
- **이 모듈의 역할**: 예방형 인지·언어 그룹 프로그램의 **성과 측정 도구**. 검사자(언어재활사·학생)가 어르신과 1:1로 실시하며 결과를 입력·저장하고 사전/사후를 비교.
- **중요 성격**: 진단·선별 도구가 아님. 정상군은 천장효과가 크므로 **언어·정서·지연회상 등 변화 민감 지표** 중심으로 본다.

---

## 2. 기술 스택 / 실행

- 현재 파일: 단일 **React 컴포넌트** (`voicelab_eval_module.jsx`), default export `App`.
- 의존성: `react`, `recharts`. 스타일은 **Tailwind CSS** 유틸리티 클래스.
- 폰트: Pretendard / Malgun Gothic (한글). 어르신 대상이라 **큰 글씨·큰 터치 영역** 유지가 원칙.
- 데모용 저장소: `window.storage` (아래 4번 참고). Claude Code 환경에서는 실제 백엔드/DB로 교체.
- 권장 셋업: Vite + React + Tailwind. (또는 voicelab 허브 기존 스택에 맞춰 이식)

## 3. 평가 배터리 (진행 순서 고정)

| # | 모듈 key | 이름 | 점수(max) | 채점 | 라이선스 |
|---|---|---|---|---|---|
| 1 | `cog_reg` | 인지 — 지남력·단어 외우기(등록) | 13 (지남력10+즉시기억3) | 검사자(O/X) | 자체 ✅ |
| 2 | `naming` | 이름대기 (15개) | 15 | 검사자(O/C/X) | 자체 ✅ |
| 3 | `cog_recall` | 인지 — 지연회상·재인 | 6 (회상3+재인3) | 회상=사람, 재인=자동 | 자체 ✅ |
| 4 | `fluency` | 생성이름대기(의미 동물 / 음소 ㄱ) | 30 | 검사자(타이머+카운터) | 공용 패러다임 ✅ |
| 5 | `digit` | 숫자 외우기(바로/거꾸로) | 13 | 검사자(최장 자릿수) | 공용 ✅ |
| 6 | `discourse` | 담화(이야기 다시말하기) | 10 | 사람(정보단위 체크) | 자체 ✅ |
| 7 | `gds` | 우울 SGDS-K (15문항 예/아니오) | 15 (낮을수록 좋음, 절단점 8) | 자동 | ⚠️ 공식문항 교체 |
| 8 | `qol` | 삶의 질 WHOQOL-BREF (26문항) | 영역 0~100 | 자동(영역점수) | ⚠️ WHO 허가 후 |

**인지 분할 흐름(핵심)**: 단어 *등록*은 1단계(`cog_reg`), 이름대기(2단계)가 **간섭과제**, *지연회상·재인*은 3단계(`cog_recall`). 등록↔회상 사이에 지연을 두는 임상 표준 설계. 순서를 바꾸지 말 것.

각 모듈은 `onDone({ score, max, detail, flags, lowerBetter? })` 형태로 결과를 올려보냄.
- `detail`: 화면 표시용 객체 (예: `{지남력:"8/10", 즉시기억:"3/3"}`)
- `flags`: 경고 문자열 배열 (예: 우울≥8, 지연회상≤1)
- `lowerBetter`: true면 점수가 낮을수록 호전 (현재 `gds`)

## 4. 데이터 모델

세션 객체:
```js
{
  id, name, age, edu, sex,
  timepoint: "pre" | "mid" | "post",
  date: "YYYY-MM-DD",
  results: { [moduleKey]: { score, max, detail, flags, lowerBetter? } }
}
```
저장 키: `voicelab:eval:{대상자ID}:{timepoint}`

**백엔드 연동 지점**: 파일 상단의 `saveSession()` / `listSessions()` **두 함수만** voicelab 허브 API로 교체하면 됨. 나머지 UI는 손대지 않아도 됨.

## 5. ⚠️ 검사도구 라이선스 제약 (절대 위반 금지)

리서치로 확인된 사항. 정식 운영 전 반드시 지킬 것:

- **K-MMSE**: 유료 라이선스(PAR/인싸이트). **앱에 내장 금지.** → 자체 지남력·기억 과제로 대체함(현재 구현).
- **CIST**: 보건복지부 저작권. **전산(웹) 평가 불가, 배포·인터넷 노출 금지.** 앱에 못 넣음. 정식 선별/진단이 필요한 대상자는 **치매안심센터에서 종이 CIST로** 받게 연계.
- **SGDS-K**: GDS 원판=공유저작물, 한국판=대한치매학회 배포. **사용 가능하나** 코드의 임시 문항을 **대한치매학회 배포본 공식 15문항 + 출처표기**로 교체해야 정식 사용. (채점 로직·절단점 8은 그대로)
- **WHOQOL-BREF**: WHO 저작물. **WHO 허가 신청 완료(승인 대기 중).** 승인 후 who.int 공식 한국어판 26문항으로 교체. 현재 코드 문항은 임시 템플릿이며 그대로 실사용 금지. (채점 로직 0~100 영역점수는 유지)
- 이름대기·유창성·숫자외우기·담화·자체 지남력/기억: **자체 제작이라 그대로 사용 가능.**

규칙: "특정 표준화 검사의 문항 세트·규준"이 저작권 대상이지, "지남력을 묻는다/단어를 외우게 한다" 같은 **패러다임 자체는 보호 대상 아님.** 그래서 자체 과제는 OK.

## 6. 완료된 것 (현재 코드 상태)

- 7영역 8단계 순차 진행 플로우 + 진행률 표시바
- 대상자 정보 입력, 시점(사전/중간/사후) 선택
- 인지 분할(등록→이름대기→지연회상) 구현, 재인 자동채점
- 각 검사 결과 요약 화면 + flags(우울≥8 등) 경고 표시
- `window.storage` 저장 + **대상자별 사전/사후 비교**(표 + recharts 막대그래프, 최대점수 대비 %)
- 라이선스 안내 문구를 코드 주석·화면 하단에 명시

## 8. 설계 원칙 / 주의사항

- **검사자-실시형**: 어르신이 혼자 하는 게 아니라 검사자가 함께 실시·입력. 버튼/글씨 크게, 한 화면에 과제 하나.
- **지남력 단서 차단**: 화면·기기에 날짜/시간/장소가 노출되면 지남력 측정 무효. UI에서 시계·날짜 숨김 유지.
- **변화 추적이 목적**: 규준·진단 라벨을 붙이지 말 것. flags는 "연계 권장" 수준의 안내까지만.
- **위험 신호 연계**: 우울 SGDS-K ≥ 8, 지연회상 현저 저하 등은 결과화면에 표시하고 **치매안심센터 등 연계** 운영지침과 연결.
- **단서 위계(언어 과제 채점 시)**: 기다림 → 의미단서 → 음소단서 → 보기. 대신 답해주지 않음. 이름대기 C(단서후정반응) 처리와 일치.
