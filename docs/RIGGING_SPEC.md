# 조음기관 3D 리깅 사양서 v2.3 (Articulator Rig Spec v2.3)

> 대림대학교 Voice Lab · 조음기관 인터랙티브 3D 애니메이션 모듈용
> 목적: 한국어 **모든 자음(19개) · 단모음(10개) · 이중모음(11개)** 을 시각적으로
> 실현할 수 있도록 두부 3D 모델을 리깅한다. 결과물은 웹(three.js)에서
> 블렌드셰이프 가중치 + 본 회전 + 공기 흐름(particle) 시각화로 실시간 제어된다.

---

## 1. 산출물 (Deliverables)

- **`*.glb` (glTF 2.0 binary)** — 최종 포맷. (FBX/Maya로 작업해도 export는 glb)
- 텍스처 포함(embedded). 단위 **meter**, **Y-up**.
- **`*.mb` (Maya 원본)** — 수정 대비 동봉.
- 리깅 전 **중립(rest) 포즈** + 아래 정의된 **12개 블렌드셰이프 타깃**.
- (선택) **공기 흐름 출구 마커**(빈 노드) — 좌표만 텍스트로 전달해도 무방.
- **검수용 영상**: 각 타깃 0→1 변형 시연 (타깃당 2~3초).

---

## 2. 모델 구조 — 풀 3D 필수

**중요**: 사지털 단면이 아니라 **풀 3D(360°)** 로 만들어 주세요. 사용자가
마우스로 회전해 **측면·정면·위·아래** 모두 볼 수 있어야 합니다. 이유:

- **마찰음 /ㅅ ㅆ/, 파찰음 /ㅈ ㅊ ㅉ/**: 혀 표면 중앙의 세로 홈(median sulcus)이
  **정면·위 시점에서 보여야** 학습자가 이해 가능.
- **설측음 /ㄹ/**: 혀 **양옆 측면 통로**가 **정면 시점에서 보여야** 함.
- **공기 흐름**: 중앙(구강) / 양옆(설측) / 비강 / 입술 출구를 시각적으로 보여줘야 함.

### 2.1 좌표 / 방향 규약
- **Y+ = 위(superior)**, **+Z 또는 +X = 정면(anterior, 코·입술 방향)** — 작업 후 명시.
- 원점은 구강 중앙 부근 권장.
- 메시는 1개든 여러 개든 무방 — **스키닝/모프만 정상**이면 됨.
- 토폴로지: 모든 블렌드셰이프는 **동일 정점 순서**(베이스와 동일 토폴로지) 공유.

## 3. 리깅 방식

| 부위 | 권장 방식 | 비고 |
|------|-----------|------|
| 턱(하악) | **블렌드셰이프** | `jaw_open` 하나로 회전 개구 표현 |
| 입술(상·하) | **블렌드셰이프** | 폐쇄/원순/펼침 3종 |

## 4. 블렌드셰이프 타깃 — 총 12개

블렌드셰이프는 **0=중립, 1=완전 변형**으로 정규화. 모든 타깃은
**상호 가산(addition) 가능**해야 함 (예: `lips_closed`+`velum_open` 동시 적용 = ㅁ).

### 혀 (Tongue) — 7개
| # | 타깃 이름 | 동작 | 한국어 음소 활용 예 |
|---|-----------|------|--------------------|
| 1 | `tongue_tip_up` | 혀끝이 치경융기(윗니 뒤)에 닿도록 상승 | ㄷ ㅌ ㄸ ㄴ ㄹ |
| 2 | `tongue_tip_down` | 혀끝 하강(중립 아래) | 보조용 |
| 3 | `tongue_front_up` | 혀 앞날(blade)이 경구개로 상승 | ㅈ ㅊ ㅉ, 모음 ㅣ ㅔ ㅐ |
| 4 | `tongue_back_up` | 혀뒤(dorsum)가 연구개로 상승·접촉 | ㄱ ㅋ ㄲ ㅇ(받침), 모음 ㅜ ㅡ ㅗ |
| 5 | `tongue_retract` | 혀 전체 후방 이동(인두 쪽) | 후설 저모음 ㅏ |
| 6 | `tongue_groove` | 혀 표면 정중앙 **세로 홈** (위·정면에서 보임) | ㅅ ㅆ, ㅈ ㅊ ㅉ |
| 7 | `tongue_lateral_channel` | **혀 양옆이 상승해 윗어금니에 접촉**, 중앙은 낮아져 좌우로 공기 통로 형성 (정면에서 보임) | ㄹ (설측음) |

### 연구개 (Velum) — 1개
| # | 타깃 이름 | 동작 |
|---|-----------|------|
| 8 | `velum_open` | 연구개·구개수 하강(비강 통로 개방). 0=거상, 1=하강 |

### 턱 (Jaw) — 1개
| # | 타깃 이름 | 동작 |
|---|-----------|------|
| 9 | `jaw_open` | 하악 회전 개구 (자연스러운 TMJ 회전, 최대 약 18–22°에 해당) |

### 입술 (Lips) — 3개
| # | 타깃 이름 | 동작 |
|---|-----------|------|
| 10 | `lips_closed` | 두 입술 폐쇄(양순 폐쇄) |
| 11 | `lips_round` | 원순 — 입술 앞으로 모음 |
| 12 | `lips_spread` | 좌우로 펼침 |

## 5. 한국어 음소 → 타깃 매핑 (전부 실현 가능)

리거 검수 기준 + 코드에서 그대로 적용할 가중치.

### 5.0 조음 원리 — 접촉 vs 좁힘 vs 개방 (중요)

같은 `tongue_tip_up` 블렌드셰이프라도 **가중치에 따라 조음 방법이 달라집니다**.
리거는 0~1 가산이 자연스럽게 이어지도록만 만들어 주시면, 음운 차이는 코드에서 가중치로 표현합니다.

| 가중치 범위 | 조음 상태 | 해당 음소 유형 |
|------------|----------|---------------|
| **= 1.0** | **완전 접촉** (혀가 치조에 닿아 폐쇄/유지) | 파열음 ㄷ ㅌ ㄸ, 비음 ㄴ, 설측음 ㄹ, 탄설음 ㄹ |
| **0.7 ~ 0.85** | **좁힘 형성** (혀가 치조 근처로 올라가되 **닿지 않고** 좁은 간격 유지 → 난기류) | 마찰음 ㅅ ㅆ |
| **0 ~ 0.3** | 거의 중립 (개방) | 모음, 후음 ㅎ |

마찬가지로:
- `tongue_back_up` = 1.0 → 완전 접촉 (파열음 ㄱ ㅋ ㄲ, 비음 ㅇ) / 0.3~0.6 → 좁힘 없는 거상 (후설 모음)
- `lips_closed` = 1.0 → 완전 폐쇄 (양순음) / 0.0 → 개방

### 5.1 자음 — 조음 위치·방법별 그룹 (11그룹)

조음기관 시각화에서 같은 위치·방법의 자음들은 **동일한 형태**를 가집니다. (평음/격음/경음 차이는 성문 긴장도·VOT에서 오며 시각적으로는 구분이 어려워 동일 그룹으로 묶습니다.)

| 음소 그룹 | IPA | 조음위치 · 방법 | 주요 타깃 (가중치) |
|-----------|-----|----------------|--------------------|
| ㅂ · ㅍ · ㅃ | /p/ /pʰ/ /p͈/ | 양순 파열 | `lips_closed`(1) |
| ㅁ | /m/ | 양순 비음 | `lips_closed`(1) + `velum_open`(1) |
| ㄷ · ㅌ · ㄸ | /t/ /tʰ/ /t͈/ | 치조 파열 | `tongue_tip_up`(1) + `jaw_open`(0.2) |
| ㄴ | /n/ | 치조 비음 | `tongue_tip_up`(1) + `velum_open`(1) |
| ㅅ · ㅆ | /s/ /s͈/ | 치조 마찰 | `tongue_tip_up`(**0.75~0.82 — 닿지 않음, 좁힘만**) + `tongue_groove`(1) + `jaw_open`(0.3) |
| ㅈ · ㅊ · ㅉ | /tɕ/ /tɕʰ/ /tɕ͈/ | 치경경구개 파찰 | `tongue_front_up`(1 — **폐쇄 시점**) + `tongue_groove`(0.5 — 개방 시) |
| ㄱ · ㅋ · ㄲ | /k/ /kʰ/ /k͈/ | 연구개 파열 | `tongue_back_up`(1) |
| ㅇ (받침) | /ŋ/ | 연구개 비음 | `tongue_back_up`(**1.0 — 완전 접촉**) + `velum_open`(1) |
| ㄹ (탄설) | /ɾ/ | 치조 탄설 (모음 사이) | `tongue_tip_up`(**1.0 — 순간 접촉**) [짧게 반복] |
| ㄹ (설측) | /l/ | 치조 설측 (어말/겹침) | `tongue_tip_up`(**1.0 — 접촉 유지**) + `tongue_lateral_channel`(1) |
| ㅎ | /h/ | 성문 마찰 | `jaw_open`(0.3) — 구강 변형 거의 없음 (**다음 모음 형태를 따름**, 동시조음) |

### 5.2 단모음 (10개)

| 모음 | IPA | 혀높이/위치/원순 | 주요 타깃 (가중치) |
|------|-----|-------------------|--------------------|
| ㅏ | /a/ | 저·중앙/후, 평순 | `jaw_open`(0.8) + `tongue_retract`(0.2) |
| ㅓ | /ʌ/ | 중·후, 평순 | `jaw_open`(0.5) + `tongue_retract`(0.2) |
| ㅗ | /o/ | 중·후, 원순 | `jaw_open`(0.3) + `lips_round`(0.7) + `tongue_back_up`(0.4) |
| ㅜ | /u/ | 고·후, 원순 | `lips_round`(1) + `tongue_back_up`(0.6) |
| ㅡ | /ɯ/ | 고·후, 평순 | `tongue_back_up`(0.5) (입술은 **중립** — `lips_round`도 `lips_spread`도 0; ㅜ와의 핵심 차이) |
| ㅣ | /i/ | 고·전, 평순 | `tongue_front_up`(0.8) + `lips_spread`(0.6) |
| ㅔ | /e/ | 중·전, 평순 | `tongue_front_up`(0.4) + `lips_spread`(0.3) + `jaw_open`(0.3) |
| ㅐ | /ɛ/ | 개중·전, 평순 | `tongue_front_up`(0.3) + `lips_spread`(0.3) + `jaw_open`(0.4) |
| ㅚ | /ø/→[we] | 현대 한국어에서 이중모음화 | **ㅜ → ㅔ** 보간 (참고 5.3) |
| ㅟ | /y/→[wi] | 현대 한국어에서 이중모음화 | **ㅜ → ㅣ** 보간 (참고 5.3) |

> ㅐ /ɛ/와 ㅔ /e/는 현대 화자(특히 젊은층)에서 [e̞]로 합류하는 추세 — 가중치는 유사하게 두어도 무방.

### 5.4 음성학적 미세 변형 / Allophones (참고)

다음은 같은 음소가 환경에 따라 약간 다르게 실현되는 경우입니다. **새 블렌드셰이프는 필요 없고**, 기존 타깃 가중치를 코드에서 환경별로 조정하면 됩니다.

| 환경 | 변형 | 가중치 조정 |
|------|------|-------------|
| **받침 위치 폐쇄음·비음** (ㅂ/ㄷ/ㄱ + ㅁ/ㄴ/ㅇ) | **미파열**(unreleased) — 폐쇄 유지, 입쪽으로 release(개방 분출) 없음. 비음은 폐쇄 유지 + 비강으로만 기류 지속 | 해당 조음 타깃(`lips_closed`/`tongue_tip_up`/`tongue_back_up`) **유지**, `jaw_open` 0 유지, release 단계 생략 |
| **ㅅ + ㅣ** | /ɕ/로 **구개음화** | `tongue_tip_up`(0.55) + `tongue_front_up`(0.5) + `tongue_groove`(1) (혀가 더 뒤·위로) |
| **ㅈ ㅊ ㅉ + ㅣ** | 더 깊은 구개음화 | `tongue_front_up`(1) 유지 + 모음 ㅣ로 자연 전이 |
| **ㅎ + 모음** | **동시조음**: 다음 모음 형태를 미리 가짐 | 후속 모음의 타깃을 미리 부분 적용 (0.3~0.5) |
| **ㄴ + ㅣ** | 약한 구개음화 [ɲ] | `tongue_tip_up`(0.85) + `tongue_front_up`(0.3) |
| **ㄹ + ㅣ** | 약한 구개화 설측 [ʎ] | 설측음 ㄹ + `tongue_front_up`(0.2) |
| **모음 사이 ㅎ** | 약화·삭제 가능 | 가중치 낮춤 또는 생략 |

## 6. 공기 흐름(Airflow) 출구 마커 — (선택)

웹 앱에서 음소별로 공기 입자가 어디로 나가는지 시각화합니다. 다음 위치에
**빈 노드(empty / locator)** 가 모델에 포함되면 코드에서 자동 인식합니다.
어려우시면 좌표만 별도 문서로 알려주셔도 됩니다.

> **개념 정리**: 아래 마커는 **공기가 최종적으로 나오는 출구**입니다. 음소 종류에 따라:
> - **마찰음 ㅅ ㅆ ㅎ**: 좁힘 부위에서 **난기류 발생** → 입 앞쪽(중앙)으로 지속 배출 → `airflow_oral_central` / `airflow_labial`
> - **설측음 ㄹ**: 혀끝은 닿아 있고 양옆으로 흐름 → `airflow_oral_left` / `airflow_oral_right`
> - **파열음 ㅂ/ㄷ/ㄱ**: 폐쇄 → 개방 순간 **파열음 분출** → 폐쇄가 풀린 위치에서 앞으로 → 최종 `airflow_labial`
> - **비음 ㅁ ㄴ ㅇ**: 구강 폐쇄 + 연구개 하강 → 비강 경로 → `airflow_nasal_l/r`

| 마커 이름 | 위치 | 활용 음소 |
|-----------|------|-----------|
| `airflow_oral_central` | 혀 중앙 홈 출구(앞날과 치경 사이) | ㅅ ㅆ ㅈ ㅊ ㅉ |
| `airflow_oral_left` | 혀 왼쪽 측면 출구(어금니 옆) | ㄹ (설측) |
| `airflow_oral_right` | 혀 오른쪽 측면 출구(어금니 옆) | ㄹ (설측) |
| `airflow_labial` | 입술 사이 정중앙 | ㅂ ㅍ 개방 시 모든 모음 |
| `airflow_nasal_l` | 왼쪽 콧구멍 | ㅁ ㄴ ㅇ |
| `airflow_nasal_r` | 오른쪽 콧구멍 | ㅁ ㄴ ㅇ |

## 7. 검수 체크리스트 (납품 전 확인)

- [ ] glb 단독으로 열었을 때(예: gltf-viewer.donmccurdy.com) 각 타깃 0→1 시 의도대로 변형되는가.
- [ ] 한 타깃을 움직일 때 **무관한 부위가 늘어나거나 따라오지 않는가** (가중치 누수 없음).
- [ ] `lips_closed` 시 두 입술이 **틈 없이** 닿는가.
- [ ] `velum_open` 시 비강 통로가 실제로 열리는가(구개수 하강).
