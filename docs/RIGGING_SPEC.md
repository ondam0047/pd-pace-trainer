# 조음기관 3D 리깅 사양서 (Articulator Rig Spec)

> 대림대학교 Voice Lab · 조음기관 시각화 모듈용
> 목적: 정중시상(midsagittal) 두부 모델의 **조음기관(혀·연구개·턱·입술)** 을
> 음소 단위로 정확히 움직일 수 있도록 리깅한다. 결과물은 웹(three.js)에서
> 본 회전 / 블렌드셰이프 가중치로 실시간 제어된다.

---

## 1. 산출물 (Deliverables)

- **`*.glb` (glTF 2.0 binary)** — 최종 포맷. (FBX/Blender로 작업하더라도 export는 glb)
- 텍스처 포함(embedded). 단위 **meter**, **Y-up**.
- 리깅 전 **중립(rest) 포즈** 1개 + 아래 정의된 **가동 타깃** 일체.
- (선택) 음소별 애니메이션 클립. 없으면 코드에서 타깃을 조합해 생성한다.

---

## 2. 좌표 / 방향 규약

- **Y+ = 위(superior)**, **Z+ = 정면(앞, anterior, 코·입술 방향)** 권장.
  - 다르게 작업했다면 "정면이 어느 축인지" 반드시 문서로 명시.
- 원점은 구강 중앙 부근 권장(엄격하지 않음). 스케일·정면축만 명확하면 됨.
- 메시는 **1개든 여러 개든 무방** — 스키닝/모프만 정상이면 됨.
- 토폴로지: 모든 블렌드셰이프는 **동일 정점 순서(베이스와 동일 토폴로지)** 를 공유.

---

## 3. 리깅 방식

| 부위 | 권장 방식 | 비고 |
|------|-----------|------|
| 턱(하악) | **본(skin)** | TMJ(귀 앞)에 회전 피벗 1개 |
| 입술(상·하) | **본 또는 블렌드셰이프** | 폐쇄/원순/펼침 |
| 혀 | **블렌드셰이프(모프 타깃) 강력 권장** | 끝말림·홈·전후 등 미세 변형은 본보다 모프가 깔끔 |
| 연구개(연구개+구개수) | **블렌드셰이프 또는 본 1개** | 하강(비음)/거상 |

> 혼합 가능: 턱·입술=본, 혀·연구개=블렌드셰이프. 전부 블렌드셰이프여도 무방.

---

## 4. 필수 가동 타깃 (정확히 이 이름으로)

블렌드셰이프는 **0=중립, 1=완전 변형**으로 정규화. 본은 해당 동작의 회전축/한계각을 문서화.

### 혀 (Tongue) — 블렌드셰이프 권장
| 타깃 이름 | 동작 |
|-----------|------|
| `tongue_tip_up` | 혀끝이 치경융기(윗니 뒤)에 닿도록 상승 |
| `tongue_tip_down` | 혀끝 하강(중립 아래) |
| `tongue_front_up` | 혀 앞날(blade)이 경구개로 상승 |
| `tongue_back_up` | 혀 뒤(dorsum)가 연구개로 상승·접촉 |
| `tongue_retract` | 혀 전체 후방 이동(인두 쪽) |
| `tongue_groove` | 혀 중앙 세로 홈 형성(마찰음 /s/ 등) |

### 연구개 (Velum)
| 타깃 이름 | 동작 |
|-----------|------|
| `velum_open` | 연구개·구개수 **하강**(비강 통로 개방=비음). 0=거상(구강), 1=하강(비강) |

### 턱 (Jaw)
| 타깃/본 | 동작 |
|---------|------|
| `jaw_open` | 하악 회전 개구. (본이면 TMJ 피벗 회전, 최대 약 18–22°) |

### 입술 (Lips)
| 타깃 이름 | 동작 |
|-----------|------|
| `lips_closed` | 두 입술 폐쇄(양순음 /p,b,m/) |
| `lips_round` | 원순(앞으로 모음, /u,o/) |
| `lips_spread` | 좌우로 펼침(/i,e/) |

> 모든 타깃은 **상호 가산(addition) 가능**해야 함 (예: `lips_closed`+`velum_open` 동시 적용 = ㅁ).

## 5. 한국어 음소 → 타깃 매핑 (검수 기준)

리거가 "이 조합이 자연스러운가"를 확인하는 용도. 숫자는 대략적 가중치(0–1).

### 자음
| 음소 | 조음위치/방법 | 주요 타깃(가중치) |
|------|----------------|-------------------|
| ㅂ·ㅍ·ㅃ | 양순 파열 | `lips_closed`(1) |
| ㅁ | 양순 비음 | `lips_closed`(1) + `velum_open`(1) |
| ㄷ·ㅌ·ㄸ | 치조 파열 | `tongue_tip_up`(1) + `jaw_open`(0.2) |
| ㄴ | 치조 비음 | `tongue_tip_up`(1) + `velum_open`(1) |
| ㅅ·ㅆ | 치조 마찰 | `tongue_tip_up`(0.6) + `tongue_groove`(1) + `jaw_open`(0.3) |
| ㅈ·ㅊ·ㅉ | 치경경구개 파찰 | `tongue_front_up`(1) + `tongue_groove`(0.5) |
| ㄱ·ㅋ·ㄲ | 연구개 파열 | `tongue_back_up`(1) |
| ㅇ(받침) | 연구개 비음 | `tongue_back_up`(0.9) + `velum_open`(1) |
| ㄹ | 치조 탄설/설측 | `tongue_tip_up`(0.8) |
| ㅎ | 성문 마찰 | `jaw_open`(0.3) (구강 타깃 거의 없음) |

### 모음 (단모음)
| 모음 | 혀높이/위치/원순 | 주요 타깃(가중치) |
|------|------------------|-------------------|
| ㅏ | 저·후, 평순 | `jaw_open`(0.8) |
| ㅓ | 중·후, 평순 | `jaw_open`(0.5) |
| ㅗ | 중·후, 원순 | `jaw_open`(0.4) + `lips_round`(0.7) + `tongue_back_up`(0.3) |
| ㅜ | 고·후, 원순 | `lips_round`(1) + `tongue_back_up`(0.6) |
| ㅡ | 고·후, 평순 | `tongue_back_up`(0.5) |
| ㅣ | 고·전, 평순 | `tongue_front_up`(0.8) + `lips_spread`(0.6) |
| ㅔ·ㅐ | 중·전, 평순 | `tongue_front_up`(0.4) + `lips_spread`(0.3) + `jaw_open`(0.3) |

## 6. 검수 체크리스트 (납품 전 확인)

- [ ] glb 단독으로 열었을 때(예: https://gltf-viewer.donmccurdy.com) **각 타깃을 0→1** 했을 때 의도대로 변형되는가.
- [ ] 한 타깃을 움직일 때 **무관한 부위가 늘어나거나 따라오지 않는가** (가중치 누수 없음).
- [ ] `lips_closed` 시 두 입술이 **틈 없이** 닿는가.
- [ ] `velum_open` 시 비강 통로가 실제로 열리는가(구개수 하강).
- [ ] `tongue_tip_up`/`tongue_back_up` 접촉 지점이 해부학적으로 맞는가(치경 vs 연구개).
- [ ] 두 타깃 **동시 적용**(예: ㅁ = lips_closed + velum_open)이 깨지지 않는가.
- [ ] 텍스처/머티리얼이 glb에 포함되어 있는가.
- [ ] 정면 축·스케일이 문서에 명시되어 있는가.

## 7. 웹 연동 (참고 — 리거가 알 필요는 없음)

- three.js / React Three Fiber에서 `gltf.scene` 로드 후
  - 블렌드셰이프: `mesh.morphTargetDictionary[name]` → `mesh.morphTargetInfluences[i] = 0..1`
  - 본: `scene.getObjectByName(boneName).rotation/position`
- 위 **타깃 이름을 그대로** 코드에서 호출하므로, 철자/대소문자를 정확히 맞춰 주세요.
- 이름이 다르면 매핑만 조정하면 됩니다(동작엔 문제 없음).
