# 이름대기 그림카드 — 사진 자산

이 폴더에 사진 파일을 두면 `components/voicelabEval/namingCards.jsx` 의
`NamingCard` 가 자동으로 SVG 대신 사진을 보여줍니다.

## 파일명 규칙

`{slug}.{ext}` 형식. 슬러그(영문)는 아래 매핑, 확장자는 `webp` → `jpg` →
`jpeg` → `png` 순으로 시도해서 처음 로드되는 걸 씁니다.

| 항목 | 슬러그 (파일명) |
| --- | --- |
| 사과 | `apple` |
| 시계 | `clock` |
| 우산 | `umbrella` |
| 안경 | `glasses` |
| 가방 | `bag` |
| 주전자 | `kettle` |
| 빗자루 | `broom` |
| 돋보기 | `magnifier` |
| 다리미 | `iron` |
| 자전거 | `bicycle` |
| 코끼리 | `elephant` |
| 절구 | `mortar` |
| 호미 | `hoe` |
| 부채 | `fan` |
| 가위 | `scissors` |

예: `apple.webp`, `kettle.jpg`, `hoe.png` …

## 권장 사양

- **해상도**: 600×600 ~ 1024×1024 정사각 비율 권장 (어르신 화면에 풀스크린으로
  띄울 때 픽셀화 안 되게).
- **포맷**: WebP 최우선(가벼움), 안 되면 JPG. PNG 는 투명 배경 필요 시.
- **배경**: 흰색 또는 단색 배경(검사 자극은 단순할수록 좋음).
- **단일 대상**: 한 컷에 한 사물만. 손/배경 사물 등 잡요소 제거.
- **각도**: 어르신이 일상에서 보는 전형적 시점(정면 또는 약간 위). 너무 추상적
  앵글 피하기.
- **한국 토속물(절구·호미·부채)** 은 한국형 모양인지 반드시 확인. 서양 mortar
  나 서양식 hand-fan 사진 쓰지 말 것.

## 라이선스

본 폴더에 두는 사진은 **반드시** 다음 중 하나여야 합니다:

1. **CC0 / Public Domain** (Wikimedia Commons 의 PD-self 등)
2. **CC-BY** (저작자 표기 — `LICENSES.md` 에 출처/저작자/라이선스 명시 필수)
3. **자체 촬영** (사업단 자산)
4. **유료/라이선스 구매본** (라이선스 증빙 보관 + `LICENSES.md` 메모)

K-MMSE/CIST 의 자극 이미지처럼 라이선스가 걸린 표준화 검사 도구의 그림을 그대로
복제하면 안 됩니다 (HANDOFF §5 참고).

## 신규 사진 추가 시 체크리스트

- [ ] 파일명 슬러그 규칙 준수
- [ ] 정사각 / 단일 대상 / 흰색 배경
- [ ] 라이선스 확인 → `LICENSES.md` 에 기록
- [ ] 추가 후 `/eval` 페이지에서 이름대기 진입 → 해당 항목 ‘보여드리기’ 로 확인

사진이 없으면 자동으로 자체 SVG 일러스트가 표시됩니다.
